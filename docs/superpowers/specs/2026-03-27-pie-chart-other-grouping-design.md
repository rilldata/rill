# Pie Chart "Other" Grouping + Custom Tooltips

**Linear issue**: [APP-818](https://linear.app/rilldata/issue/APP-818)
**Branch**: `ericokuma/app-818-canvas-pie-chart-unreadable-with-many-slices-add-other`
**Date**: 2026-03-27

## Problem

Canvas pie/donut charts render all slices regardless of count, making them unreadable with many categories. No hover percentage is shown. Reported by Kevel.

## Design

### 1. Data Transformation: "Other" Grouping

A new `computeVisibleSlices` function runs at render time on fetched data, between `CircularChartProvider` and `generateVLPieChartSpec`.

**Location**: `web-common/src/features/components/charts/circular/other-grouping.ts`

**Algorithm**:
- If the editor set an explicit `limit` in YAML, use it directly: show top N slices, group the rest into "Other"
- If no explicit limit, use the dynamic threshold: add slices (sorted by value descending) until the remainder is ≤ 20% of total OR 10 slices are reached (whichever comes first)
- If ≤ 5 categories total, skip grouping entirely

**Data flow**:
```
API (top 20 items) → CircularChartProvider
    → computeVisibleSlices()
        → visible slices (passed to Vega-Lite as data)
        → otherSliceMetadata (stored for tooltip breakdown)
    → generateVLPieChartSpec()
```

**Output shape**:
```typescript
interface OtherGroupingResult {
  // Data rows to pass to Vega; "Other" row has __isOther: true
  visibleData: Record<string, unknown>[];
  // Items grouped into "Other", for tooltip breakdown
  otherItems: Array<{ label: string; value: number }> | null;
  // Total across ALL items (for percentage calculation)
  total: number;
}
```

**YAML config**:
- `showOther` (boolean, default: `true`) on the `color` field config. When `true` and no explicit `limit`, use the dynamic algorithm. When `false`, truncate at `limit` with no "Other" slice.
- Existing `limit` property: when explicitly set, bypasses dynamic algorithm. When omitted, dynamic algorithm applies.

### 2. Vega-Lite Spec Changes

**File**: `web-common/src/features/components/charts/circular/pie.ts`

Changes to `generateVLPieChartSpec`:
- Accept the `OtherGroupingResult` as input (data already grouped)
- Conditional color encoding: "Other" row uses a resolved muted color (`--muted` CSS variable); named slices use the normal color scale
- Conditional stroke on arc mark: "Other" row gets `strokeDash: [4, 3]` with `stroke: var(--border)`, 1px
- Remove native tooltip encoding (tooltip will be handled by custom Svelte component)

**Conditional encoding approach**: Use Vega-Lite `condition` expressions on the color and strokeDash channels, testing `datum.__isOther === true`.

### 3. Custom Tooltip Component

**File**: `web-common/src/features/components/charts/circular/PieChartTooltip.svelte`

A Svelte component that renders tooltips for pie chart slices, replacing Vega-Lite's native tooltip.

**Integration**: The existing `VegaLiteRenderer` already supports a `tooltipFormatter` prop that overrides Vega's native tooltip handler. For pie/donut charts, we provide a custom formatter that:
1. Receives the hovered datum from Vega (including the `__isOther` flag and slice field values)
2. Renders the appropriate HTML (regular or "Other" layout) as a string
3. Returns it to the existing `VegaLiteTooltipHandler`, which handles positioning via `placeElement`

This means we don't need a separate Svelte component mount/unmount cycle. The tooltip HTML is generated by a formatter function and injected into the existing tooltip DOM element. This approach reuses the existing tooltip infrastructure while giving us full control over layout.

**Regular slice tooltip layout**:
```
┌─────────────────────────────────┐
│  ● Display Ads · $12,450 · 34.2%  │
└─────────────────────────────────┘
```

Styling:
- Container: `bg-popover text-popover-foreground shadow-sm rounded-md p-2`
- Color dot: 8px circle matching slice color
- Name: `text-sm font-medium text-foreground`
- Value + percentage: `text-sm text-muted-foreground`, separated by `·`
- Percentage: always 1 decimal place, calculated against total of ALL data (not just visible)

**"Other" slice tooltip layout (mini-leaderboard)**:
```
┌──────────────────────────────────────┐
│  Other · $5,230 · 14.4%             │
│  ─────────────────────────────────── │
│  Native Ads              $1,820  5.0%│
│  Video Pre-roll          $1,340  3.7%│
│  Interstitial              $890  2.5%│
│  Sponsored Content         $670  1.8%│
│  Rich Media                $510  1.4%│
│  and 8 more                          │
└──────────────────────────────────────┘
```

Styling:
- Container: `bg-popover text-popover-foreground shadow-sm rounded-md p-3 min-w-[240px] max-w-[320px]`
- Header: muted dot + "Other" label + value + percentage
- Divider: `border-b border-border my-1.5`
- Breakdown rows: flex layout, dimension label left (truncated), value + percentage right, `tabular-nums`
- Footer: `text-xs text-muted-foreground italic pt-1`, "and {N} more" (shown only when >5 items)
- Max breakdown rows shown: 5 (hardcoded)

### 4. Legend

Use Vega-Lite's built-in legend. The "Other" entry appears last (it's last in the sorted data). Its color in the legend will be the muted color from the conditional color encoding. No custom dashed-outline styling on the legend dot; the muted color alone provides sufficient differentiation.

### 5. Behavioral Details

- **No animation on slice count changes**: Default Vega-Lite behavior; no additional work.
- **Percentage calculation**: Always uses total of ALL items (including those in "Other"), not just visible slice values.
- **Dark mode**: All CSS variables (`--muted`, `--border`, `--popover`, etc.) resolve correctly per theme mode. The tooltip component uses Tailwind classes that reference these tokens.
- **Keyboard accessibility**: Deferred. The spec mentions Tab-focusable slices, but Vega-Lite canvas rendering doesn't support this natively. Would require SVG rendering or a custom keyboard layer. Out of scope for this iteration.

### 7. Out of Scope

- On-slice percentage labels (unreadable on small donuts)
- Click-to-expand "Other" into secondary ring
- `showPercent` YAML property (hover % is always on)
- Keyboard/Tab navigation for slices (Vega canvas limitation)
- Custom legend component with dashed-outline dot for "Other"

### 8. Acceptance Criteria

- [ ] Donut chart with ≤5 categories: all shown, no "Other" slice
- [ ] Donut chart with 8 categories (long-tail, top 3 = 80%): shows ~4-5 slices + small "Other"
- [ ] Donut chart with 20 categories (even distribution, ~5% each): shows 10 slices + "Other" (~50%)
- [ ] Explicit `limit: 3` in YAML: shows exactly 3 slices + "Other" regardless of distribution
- [ ] `showOther: false` with `limit: 3`: shows exactly 3 slices, no "Other"
- [ ] Hovering a named slice shows tooltip with color dot, name, formatted value, and % (1 decimal)
- [ ] Hovering "Other" shows mini-leaderboard with top 5 grouped items + "and N more"
- [ ] "Other" slice has dashed border and muted fill color
- [ ] Legend shows "Other" last with muted color
- [ ] Dark mode: all tokens resolve correctly
- [ ] No animation when slice count changes between renders
