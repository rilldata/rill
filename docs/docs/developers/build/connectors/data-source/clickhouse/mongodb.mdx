## Overview

[MongoDB](https://www.mongodb.com/) is a document-oriented NoSQL database. ClickHouse can query MongoDB collections directly using its [`mongodb()` table function](https://clickhouse.com/docs/en/sql-reference/table-functions/mongodb), pulling data into your Rill models without intermediate exports.

<ClickHousePrereq />

## Credentials

Add your MongoDB credentials to your project's `.env` file:

```bash
MONGODB_HOST=your_host
MONGODB_PORT=27017
MONGODB_DATABASE=your_database
MONGODB_USER=your_user
MONGODB_PASSWORD=your_password
```

For details on managing credentials, see [Configure Local Credentials](/developers/build/connectors/credentials).

## Model Configuration

```yaml
type: model
connector: my_clickhouse

sql: |
  SELECT *
  FROM mongodb(
    '{{ .env.MONGODB_HOST }}:{{ .env.MONGODB_PORT }}',
    '{{ .env.MONGODB_DATABASE }}',
    'my_collection',
    '{{ .env.MONGODB_USER }}',
    '{{ .env.MONGODB_PASSWORD }}',
    'id Int64, name String, created_at DateTime'
  )
```

:::info Explicit Schema Required
MongoDB is schema-less, so you must define column names and ClickHouse types in the last argument. ClickHouse maps BSON fields to the specified types. Only the fields you list will be read from MongoDB.
:::

### Common Type Mappings

| BSON Type | ClickHouse Type |
|---|---|
| String | `String` |
| 32-bit integer | `Int32` |
| 64-bit integer | `Int64` |
| Double | `Float64` |
| Boolean | `Bool` |
| Date | `DateTime` |
| ObjectId | `String` |
| Array | `Array(T)` |

:::tip Network Access
Your ClickHouse instance must be able to reach the MongoDB server. If you're using Rill-Managed ClickHouse, ensure your MongoDB database is accessible from the internet or configure appropriate network peering.
:::
