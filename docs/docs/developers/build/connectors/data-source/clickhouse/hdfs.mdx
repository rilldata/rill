## Overview

[HDFS](https://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-hdfs/HdfsDesign.html) (Hadoop Distributed File System) is a distributed storage system designed for large-scale data processing. ClickHouse can read files directly from HDFS using its [`hdfs()` table function](https://clickhouse.com/docs/en/sql-reference/table-functions/hdfs), with support for Parquet, CSV, JSON, and other formats.

<ClickHousePrereq />

## Model Configuration

### Reading Parquet Files

```yaml
type: model
connector: my_clickhouse

sql: |
  SELECT *
  FROM hdfs(
    'hdfs://namenode:9000/path/to/data/*.parquet',
    'Parquet'
  )
```

### With Explicit Schema

For formats that don't include column metadata (e.g., CSV), define the schema explicitly:

```yaml
type: model
connector: my_clickhouse

sql: |
  SELECT *
  FROM hdfs(
    'hdfs://namenode:9000/path/to/data.csv',
    'CSV',
    'id UInt64, name String, created_at DateTime'
  )
```

### Path Patterns

Use wildcards to read multiple files:

```sql
-- All Parquet files in a directory
SELECT * FROM hdfs('hdfs://namenode:9000/data/*.parquet', 'Parquet')

-- Recursive glob
SELECT * FROM hdfs('hdfs://namenode:9000/data/**/*.parquet', 'Parquet')

-- Numbered ranges
SELECT * FROM hdfs('hdfs://namenode:9000/data/part_{1..10}.parquet', 'Parquet')
```

## Supported Formats

Common format options include `Parquet`, `CSVWithNames`, `JSONEachRow`, `TSVWithNames`, and `ORC`. See the [ClickHouse formats documentation](https://clickhouse.com/docs/en/interfaces/formats) for the full list.

:::tip Network Access
Your ClickHouse instance must have network access to the HDFS NameNode and DataNodes. This typically requires ClickHouse to be deployed within the same network as your Hadoop cluster.
:::
