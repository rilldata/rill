## Overview

[Azure Blob Storage](https://learn.microsoft.com/en-us/azure/storage/blobs/storage-blobs-introduction) is Microsoft's object storage service. ClickHouse reads Azure data directly using its [`azureBlobStorage()` table function](https://clickhouse.com/docs/en/sql-reference/table-functions/azureBlobStorage), which supports Parquet, CSV, JSON, and other formats.

<ClickHousePrereq />

## Credentials

Add your Azure credentials to your project's `.env` file. You can use either a connection string or account key:

**Connection string (simplest):**

```bash
AZURE_STORAGE_CONNECTION_STRING=DefaultEndpointsProtocol=https;AccountName=myaccount;AccountKey=mykey;EndpointSuffix=core.windows.net
```

**Account name and key:**

```bash
AZURE_STORAGE_ACCOUNT=myaccount
AZURE_STORAGE_KEY=mykey
```

For details on managing credentials, see [Configure Local Credentials](/developers/build/connectors/credentials).

## Model Configuration

### Using a Connection String

```yaml
type: model
connector: my_clickhouse

sql: |
  SELECT *
  FROM azureBlobStorage(
    '{{ .env.AZURE_STORAGE_CONNECTION_STRING }}',
    'my-container',
    'path/to/data/*.parquet',
    'Parquet'
  )
```

### Using Account Key

```yaml
type: model
connector: my_clickhouse

sql: |
  SELECT *
  FROM azureBlobStorage(
    'DefaultEndpointsProtocol=https;AccountName={{ .env.AZURE_STORAGE_ACCOUNT }};AccountKey={{ .env.AZURE_STORAGE_KEY }};EndpointSuffix=core.windows.net',
    'my-container',
    'path/to/data/*.parquet',
    'Parquet'
  )
```

## Path Patterns

Use wildcards to read multiple files from a container:

```sql
-- All Parquet files in a path
SELECT * FROM azureBlobStorage(..., 'my-container', 'data/*.parquet', 'Parquet')

-- Specific partitions
SELECT * FROM azureBlobStorage(..., 'my-container', 'data/{2024,2025}/*.parquet', 'Parquet')
```
