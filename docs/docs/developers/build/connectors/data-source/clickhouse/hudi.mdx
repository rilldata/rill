## Overview

[Apache Hudi](https://hudi.apache.org/) is an open-source data lake framework that provides record-level insert, update, and delete capabilities. ClickHouse can read Hudi tables directly from S3 using its [`hudi()` table function](https://clickhouse.com/docs/en/sql-reference/table-functions/hudi).

<ClickHousePrereq />

## Credentials

Add your AWS credentials to your project's `.env` file:

```bash
AWS_ACCESS_KEY_ID=your_access_key_id
AWS_SECRET_ACCESS_KEY=your_secret_access_key
```

For details on managing credentials, see [Configure Local Credentials](/developers/build/connectors/credentials).

## Model Configuration

```yaml
type: model
connector: my_clickhouse

sql: |
  SELECT *
  FROM hudi(
    'https://my-bucket.s3.amazonaws.com/path/to/hudi/table',
    '{{ .env.AWS_ACCESS_KEY_ID }}',
    '{{ .env.AWS_SECRET_ACCESS_KEY }}'
  )
```

:::info
The `hudi()` function points to the table's root directory. ClickHouse reads the latest committed snapshot by default.
:::
