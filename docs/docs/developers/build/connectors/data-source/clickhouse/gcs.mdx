## Overview

[Google Cloud Storage](https://cloud.google.com/storage/docs) is Google's object storage service. ClickHouse accesses GCS using its [`gcs()` table function](https://clickhouse.com/docs/en/sql-reference/table-functions/gcs), which wraps the S3-compatible API. You must use **HMAC keys** rather than JSON service account credentials.

<ClickHousePrereq />

## Credentials

Add your GCS HMAC keys to your project's `.env` file:

```bash
GCS_HMAC_KEY_ID=your_hmac_key_id
GCS_HMAC_SECRET=your_hmac_secret
```

:::warning HMAC Keys Required
The `gcs()` function uses the S3-compatible API under the hood, which requires HMAC keys. JSON service account credentials are not supported. See [how to create HMAC keys](/developers/build/connectors/data-source/duckdb/gcs#how-to-create-hmac-keys-for-gcs) for setup instructions.
:::

For details on managing credentials, see [Configure Local Credentials](/developers/build/connectors/credentials).

## Model Configuration

```yaml
type: model
connector: my_clickhouse

sql: |
  SELECT *
  FROM gcs(
    'https://storage.googleapis.com/my-bucket/path/to/data/*.parquet',
    '{{ .env.GCS_HMAC_KEY_ID }}',
    '{{ .env.GCS_HMAC_SECRET }}',
    'Parquet'
  )
```

## Path Patterns

Use wildcards and globs to read multiple files:

```sql
-- All Parquet files in a directory
SELECT * FROM gcs('https://storage.googleapis.com/my-bucket/data/*.parquet', ...)

-- Specific partitions
SELECT * FROM gcs('https://storage.googleapis.com/my-bucket/data/{2024,2025}/*.parquet', ...)
```
